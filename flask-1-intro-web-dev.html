Client/Server Relationship Review

Talking Points:

  • Computers communicate with one another through the client-server model. The browser (a.k.a. client) makes a request to the server to view a website, and the server responds by sending back the corresponding files to the client.
  • Our computers make an ask of the server, and then the server sends back a response - we are going to learn how to make those "asks" with Python + HTML, and then also how to write programs that guide the "responses" with Python.

In Real Terms

  • Websites are just files your browser can read.

Talking Points:

  • Websites are just files your browser can read!
  • Different kinds of files can be retrieved from the server, including HTML + CSS files, which the client then reads and renders as a website!

What Types of Files?

HTML - .html

  • Provides website structure.

CSS - .css

  • Adds colors and fonts.

JavaScript - .js

  • Makes the website interactive.

Images, text files, etc.

  • Additional info on the page

Talking Points:

  • Note that we're going to learn HTML and CSS - don't spend a lot of time here!

Quick Review

Where are websites?

  • IP Address: The actual location of a website on the internet.
  • Google.com is a friendly name for the IP address 172.217.12.142, just like "Joe's Florist" is a friendly name for "515-115-5156".

What do websites have?

  • Websites are actually just a bunch of files - images, text, and website-specific code.
  • They're hosted on servers - all the files for Google.com are on Google's servers.
  • Your browser is the client - it asks Google for the Google.com files, so it can show them to you!

Types of Web Developers:

Front-end developer

  • HTML/CSS/JS

Back-end developer

Full stack developer

Discussion: Web App vs Website

Does anyone want to guess (or know) what the difference is?

Teaching Tips:

  • Encourage Discussion

Web App vs Website

Website:

  • Typically informational
  • Little to no interactive capabilities.
  • New York Times or a small company's website.

Web app:

  • An app hosted on the internet
  • Uses the client-server relationship to render a website.
  • Offers the user greater features then a static website.
  • A bank's web page or an auction site.

You can have a hybrid!

  • For example, a static website until the user logs in.
  • Then, a full fledged web app.

Teaching Tips:

  • Be clear about the terms "web app" and "website" and "web page"

Talking Points:

  • Note: This was taken liberally from https://modeeffect.com/key-differences-between-website-web-app/

  • Websites are typically informational in nature. Think about your favorite blog or news based site. Its primary purpose is to convey information to the end user, whether it in the form of news, like CNN, or recipes, like you’ll find on Martha Stewart.

  • As a general rule, there is little or no interaction on the part of the visitor, other than possibly submitting an email address to receive a monthly newsletter or performing a search. So the real question is, how does this apply to you?

  • Well, if you’re a local charity who wants to convey information only — a homepage, an about page, contact information, upcoming events, and maybe a description of how you’ve helped your cause — a website might be all that you need.

  • You also need to consider that many websites are actually website/web application hybrids. Your startup might provide all kinds of information to visitors, but once they register, they could have access to an integrated web application that performs a specific function
